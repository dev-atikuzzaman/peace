# নূর-এ-হেদায়াহ — Phase 1+2 (Foundation + কুরআন ট্যাব)

Next.js + Tailwind + Supabase (Auth + Realtime + RLS) দিয়ে বানানো একটা PWA।
এই ফেজে আছে: single-admin authentication (লোগোতে ট্যাপ), রিয়েল-টাইম সিংকড কুরআন আয়াত ট্যাব
(সার্চ, শেয়ার, ফেভারিট, অ্যাডমিন এডিট/ডিলিট), আর বাকি ৩টা ট্যাবের কাঠামো (Phase 3/4-এ পূর্ণ হবে)।

---

## ১. Supabase সেটআপ (৫ মিনিট)

1. তোমার Supabase প্রজেক্ট খোলো → **SQL Editor** → নতুন query
2. `supabase/schema.sql` ফাইলের পুরো কনটেন্ট পেস্ট করে **Run** চাপো
   - এটা সব টেবিল, RLS পলিসি, realtime, আর "প্রথম সাইন-আপ = অ্যাডমিন" ট্রিগার বানিয়ে দেবে
3. **Project Settings → API** থেকে কপি করো:
   - `Project URL` → এটাই `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` key → এটাই `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. **Authentication → URL Configuration** এ গিয়ে তোমার Vercel ডোমেইন
   (যেমন `https://your-app.vercel.app`) কে Site URL এবং Redirect URLs এ যোগ করো —
   না হলে ইমেইল ভেরিফিকেশন লিংক সঠিক জায়গায় নিয়ে যাবে না।

## ২. GitHub এ পুশ করা

এই পুরো ফোল্ডারের কনটেন্ট তোমার রেডি করা GitHub রিপোতে পুশ করো:

```bash
cd your-existing-repo
# এই zip এর সব ফাইল কপি করো, তারপর:
git add .
git commit -m "Phase 1+2: foundation + Quran tab"
git push
```

## ৩. Vercel এ Environment Variables বসানো

Vercel প্রজেক্ট → **Settings → Environment Variables** এ যোগ করো:

| Key | Value |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | তোমার Supabase Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | তোমার Supabase anon public key |

তারপর **Redeploy** করো (env variable বসানোর পর নতুন deploy লাগবে, পুরনো build এ কাজ করবে না)।

## ৪. প্রথমবার নিজেকে অ্যাডমিন বানানো

1. ডিপ্লয় হওয়া অ্যাপ খোলো
2. উপরের বাম দিকে লোগোতে ট্যাপ করো → "অ্যাডমিন হিসেবে সাইন-আপ" ফর্ম আসবে
3. তোমার ইমেইল ও পাসওয়ার্ড দাও → সাবমিট করো
4. Supabase থেকে পাঠানো ভেরিফিকেশন ইমেইলে ক্লিক করো
5. অ্যাপে ফিরে এসে লোগোতে আবার ট্যাপ করো → এবার "অ্যাডমিন লগ-ইন" ফর্ম দেখাবে → পাসওয়ার্ড দিয়ে ঢুকে যাও

### 🔒 গুরুত্বপূর্ণ — সাইন-আপ বন্ধ করে দাও
তুমি অ্যাডমিন হয়ে যাওয়ার পর, Supabase Dashboard → **Authentication → Sign In / Providers**
এ গিয়ে **"Allow new user signups"** অপশনটা বন্ধ করে দাও। এতে তুমি ছাড়া আর কেউ কখনো
নতুন অ্যাকাউন্ট বানাতে পারবে না — সত্যিকারের single-admin ব্যবস্থা নিশ্চিত হবে।

## ৫. PWA আইকন
`public/icons/` ফোল্ডারে `icon-192.png` ও `icon-512.png` বসাও (নিজের লোগো দিয়ে)।
এটা ছাড়াও অ্যাপ পুরোপুরি কাজ করবে, শুধু হোমস্ক্রিনে ইনস্টল করার আইকনটা ডিফল্ট দেখাবে।

## আয়াতের তথ্য যেভাবে ইনপুট দেবে (Claude টেমপ্লেট)

`data/verse-template.json` কপি করে আমাকে (Claude) বলো:
> "এই ফরম্যাটে সূরা বাকারা আয়াত ২৫৫ (আয়াতুল কুরসী) এর তথ্য দাও"

আমি সেই স্ট্রাকচারেই ভরাট করে দেব। `data/sample-surah-fatiha.json` এ একটা
যাচাই-করা উদাহরণ আছে দেখার জন্য। **তাফসীর ও হাদীস অংশ পাবলিশ করার আগে একজন
জ্ঞানী ব্যক্তির মাধ্যমে আরেকবার চেক করিয়ে নেয়াটা সবচেয়ে নিরাপদ** — বিশেষ করে
হাদীস নম্বরের নির্ভুলতার জন্য।

অ্যাপে ঢুকে অ্যাডমিন হিসেবে "+ আয়াত" বাটনে চাপলেই ফর্ম আসবে, সেখানে এই
তথ্যগুলো বসিয়ে দিলেই সাথে সাথে সব ভিউয়ারের কাছে (realtime এ) দেখা যাবে।

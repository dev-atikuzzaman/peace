# নূর-এ-হেদায়াহ — Phase 1+2+3+4 (সম্পূর্ণ) + প্রিমিয়াম থিম

Next.js + Tailwind + Supabase (Auth, Realtime, RLS, Storage) দিয়ে বানানো একটা PWA।
থিম এখন তোমার akibbd/petrohub-এর "Premium Forest" ভাষায় — গাঢ় বন-সবুজ গ্র্যাডিয়েন্ট
ব্যাকগ্রাউন্ড, মিন্ট-গোল্ড অ্যাকসেন্ট, Playfair Display হেডিং, গ্লাস-কার্ড ইফেক্ট।

## এই ডেলিভারিতে যা আছে
- ৪টা ট্যাবই সম্পূর্ণ কার্যকর: কুরআন, ৯৯ নাম, হাদীস, নোট/ফাইল
- ৯৯ নামের সবগুলো সিড করা আছে (`supabase/seed_asma_ul_husna.sql`)
- ৮০/২০ নীতিতে ১৫টা অত্যন্ত পরিচিত সহীহ হাদীস সিড করা আছে (`supabase/seed_hadith_topics.sql`)
- নোট ট্যাবে ফোল্ডার, admin/public প্রাইভেসি, ফাইল আপলোড (Supabase Storage, সাইনড URL দিয়ে)
- Single-admin auth (লোগোতে ট্যাপ), realtime সিংক সব ট্যাবে

## ২. 🎨 UI পুরো unstyled/ভাঙা দেখানোর ব্যাপারে — যা পেলাম

তোমার `peace` রিপোর zip খুঁটিয়ে দেখেছি — **`tailwind.config.js`, `postcss.config.js`,
`next.config.js`, `globals.css`, `layout.js` — সবগুলো ফাইল হুবহু ঠিক আছে**, কোনো কোড বাগ
নেই (আগের স্টাইল করা "Admin Access" মডালের স্ক্রিনশটেও এই একই কোড দিয়েই কাজ করেছিল)।
তাই এটা কোডের সমস্যা না — এটা **Vercel ডিপ্লয়মেন্ট/প্রজেক্ট সেটিংসের সমস্যা** হওয়ার
সম্ভাবনাই বেশি। যা চেক করবে:

1. Vercel প্রজেক্ট → **Settings → General → Framework Preset** এ "Next.js" লেখা আছে কিনা
   (অন্য কিছু, যেমন "Other", থাকলে CSS বিল্ড হবে না)
2. **Settings → General → Root Directory** খালি আছে কিনা (বা যেখানে `package.json` আছে
   ঠিক সেই ফোল্ডার পয়েন্ট করছে কিনা)
3. সবশেষ Deployment-এ গিয়ে **"..." মেনু → Redeploy → "Use existing Build Cache" আনচেক
   করে** — একটা ক্লিন রিবিল্ড ট্রাই করো
4. তাও ঠিক না হলে, Deployment-এর **Build Logs** এর স্ক্রিনশট আমাকে পাঠাও — ওখানেই
   আসল কারণ (যদি কোনো এরর/ওয়ার্নিং থাকে) স্পষ্ট দেখা যাবে

এই zip-টা তোমার রিপোর সব ফাইল replace করে আবার পুশ করলেই (নতুন কমিটসহ) Vercel
নতুন করে বিল্ড করবে — এটাও একবার ক্লিন রিবিল্ডের সুযোগ দেবে।

## ৩. GitHub এ পুশ করা

পুরো ফোল্ডারের কনটেন্ট রিপোতে replace করে পুশ করো:

```bash
cd your-existing-repo
# zip এর সব ফাইল কপি করো (পুরনোগুলো রিপ্লেস হবে), তারপর:
git add .
git commit -m "Phase 3+4 + premium forest theme"
git push
```

## ৪. Vercel Environment Variables (আগে করা থাকলে স্কিপ করো)

| Key | Value |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | তোমার Supabase Project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | তোমার Supabase anon public key |

## আয়াতের তথ্য ইনপুট
`data/verse-template.json` কপি করে বলো: "এই ফরম্যাটে সূরা X আয়াত Y এর তথ্য দাও"।
`data/sample-surah-fatiha.json` এ একটা যাচাই-করা উদাহরণ আছে। তাফসীর/হাদীস অংশ
পাবলিশের আগে একজন জ্ঞানী ব্যক্তির মাধ্যমে চেক করিয়ে নেয়া ভালো।

## গুরুত্বপূর্ণ নোট
- ৯৯ নাম ও ১৫টা হাদীস প্রতিষ্ঠিত সোর্স থেকে লেখা হয়েছে, তবে বড় পরিসরের কনটেন্ট
  হওয়ায় একজন আলেমের মাধ্যমে একবার রিভিউ করিয়ে নেয়া সবচেয়ে নিরাপদ।
- নোট ফাইলের প্রাইভেসি: "শুধু অ্যাডমিন" নোটের টেক্সট RLS দিয়ে সম্পূর্ণ সুরক্ষিত,
  কিন্তু সেই নোটে যদি ফাইল যুক্ত থাকে, সেই ফাইলের সাইনড-URL এর মেয়াদ ১ ঘণ্টা —
  এর মধ্যে লিংক কারো হাতে গেলে সে খুলতে পারবে। সত্যিকারের অতি-সংবেদনশীল ফাইলের
  জন্য এটা মাথায় রেখো।
